## 四、命名规范

### 文件命名

- **Java类**: PascalCase（如 `OrderController.java`）
- **Vue组件**: PascalCase（如 `OrderCard.vue`）
- **JavaScript文件**: camelCase（如 `orderService.js`）
- **样式文件**: kebab-case（如 `order-list.css`）
- **配置文件**: snake-case（如 `docker-compose.yml`）
- **文档文件**: 中文名称（如 `项目分工.md`）

### 目录命名

- **Java包**: 小写单词（如 `controller`, `service`）
- **前端目录**: kebab-case（如 `order-list`, `service-detail`）
- **功能模块**: 英文单词（如 `auth`, `catalog`, `booking`）
- **层级目录**: 功能性命名（如 `components`, `pages`, `static`）

## 七、快速开始指引

### 后端开发（人员A）

```bash

mvn spring-boot:run
# 访问 http://localhost:8080
```

### 前端开发（人员B/C/D）

## 八、注意事项

1. **文档与实现一致性**: 本目录结构基于文档设计，但当前实现可能与文档有差异，请在开发过程中逐步对齐
2. **微服务迁移**: 当前单体架构可满足MVP需求，后续按计划迁移到微服务架构
3. **前端框架迁移**: 当前HTML版本用于快速原型，后续迁移到Uni-app
4. **测试先行**: 人员D负责测试统筹，各模块开发前应先编写测试用例
5. **设计规范**: 人员D负责UI规范，人员B/C在开发前端时需遵循统一设计Token
6. **API对齐**: 人员A提供API规范，人员B/C/D开发前端时需严格遵循接口定义
